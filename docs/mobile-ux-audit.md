# 모바일 앱 UX 전수 감사 — Claude Design 핸드오프

> 작성 2026-06-20 · 대상: 현장 정비사용 안드로이드 프로토타입 (`BBD Mobile (standalone).html`, gzip 해제한 실제 React 소스 기준)
> 방법: 6개 UX 이론 렌즈로 전 화면 병렬 감사 — **닐슨 10대 휴리스틱 · 정보구조/네비게이션 · 모바일 에르고노믹스/현장성 · 비주얼 위계/게슈탈트 · 콘텐츠/상태/데이터 정직성 · 접근성(WCAG)**. raw 94건 → 13개 테마.
> 사용자: BRANCH_STAFF(정비사) 단일 — 장갑·한 손·더러운 손·직사광·약전파 현장. 본사·카운터·ADMIN은 웹 전용.

## 근본 원인 (테마를 관통하는 7축)
1. **데이터 정직성 붕괴** — 백엔드가 못 받거나 없는 걸 입력받아 버리거나 사실처럼 표시(메모 삼킴·ETA 날조·이동이력 합성·하드코딩 시간·입고후현재고 추정).
2. **네비게이션 모델 구조 결함** — 탭바 유무가 라우터가 아니라 컴포넌트 구현에 묶임 → 뒤로가기·탭·진입경로가 비결정.
3. **상태 부재** — 전 화면 로딩/에러/빈/오프라인/세션만료 없음, 쓰기는 무조건 성공 가정.
4. **죽은 컨트롤** — 동작 안 하는 가짜 어포던스(기간칩·벨·설정행·플래시).
5. **색 의미 역전** — 부족=파랑(주행동색)으로 신호색 충돌.
6. **에르고노믹스** — 44dp 미만 터치타깃 + 핵심 액션 엄지존 밖.
7. **접근성 기본기 부재** — 아이콘버튼 이름·포커스·대비·reduced-motion 전반.

가장 결함이 밀집한 화면은 **입고 스캔(ScanScreen/ReceiveForm)** — 전량수령 강제 + 메모 삼킴 + 부분수령 경로 없음이 결합해 현장 최중요 예외정보(파손·수량차이)를 구조적으로 버린다.

---

## 먼저 고칠 8가지 (Top Priorities)
1. **ReceiveForm 메모 필드 제거 or "저장 안 됨" 명시** — receive PATCH가 body를 안 받아 파손·수량 메모가 조용히 버려짐(정직성 붕괴부터 차단).
2. **SoRow의 'ETA'·부품상세 '최근 이동' 제거** — 백엔드 데이터원이 없는 날조를 진실처럼 표시.
3. **전 화면 로딩/에러/빈/오프라인 + receive 실패·409 분기 추가**, 성공 모달은 2xx 응답 후에만.
4. **탭바 노출을 라우터 단일 소스로** — 루트 4탭=항상 탭바, push 스택=풀스크린. 재고의 'back=무조건 홈' 하드코딩·헤더 우측 슬롯의 단계별 의미 변동 제거.
5. **ScanScreen '탭=스캔' 거대영역·수동입력 SKU 프리필 제거** + 스캔 직후 인식코드 확인 비트 → 우발 스캔·오입고 차단.
6. **하드코딩 시간/문자열 런타임 계산** — '오늘/어제', pwDaysAgo:198, 아바타 '정민'을 today·USER.name 기준으로.
7. **아이콘 버튼 전체 aria-label/role + :focus-visible 포커스 링** — 스크린리더·키보드 접근 복구.
8. **'부족' 색 파랑→amber 분리**(신호색 3단 정상 초록/부족 amber/없음 빨강) + 죽은 컨트롤 실연결 또는 숨김.

## P0 — 핵심 과업 차단 · 데이터 부정직 (6테마)

### P0-1. 백엔드에 없는/못 받는 데이터를 만들어 보여주거나 입력시킴
**원칙**: 현실세계 부합 + 시스템 상태 가시성(데이터 정직성)
- (a) 도착대기 SoRow **'ETA'** — sales에 배송추적·예정시각 데이터원 없음(완전 날조). 게다가 표시값(05/22)이 오늘(06/20)보다 과거라 "곧 온다" 오판.
- (b) 부품상세 **'최근 이동'** — StockMovement 조회 API 없는데 `MOVES_BY_SKU` 하드코딩, 폴백은 안전재고를 가짜 입고로 합성.
- (c) ReceiveForm **'메모(파손·수량)'** — PATCH /receive가 body를 안 받아 전송 없이 사라짐(현장 최중요 예외정보 손실).
- (d) **'입고 후 현재고 = 현재고+수량'** — 서버 미반영 클라 추정을 확정 미래값처럼 파랑 강조.
- (e) ScanScreen이 탭 위치 무관 항상 `BBD-OIL-1006`으로 resolve → 'A박스 찍고 B부품 확정' 오입고 구조.
**개선**: ETA 제거→'SO 상태: 도착 대기'. '최근 이동'은 API 없으면 섹션 제거 또는 RECEIVED SO에서 '도착 입고' 행만 파생. 메모는 저장채널 없으면 비노출(데모면 '저장 안 됨' 명시). '입고 후 현재고'는 '예상 ≈' 라벨 또는 제거, 확정은 receive 2xx/후속 /stocks 재조회로만. 스캔은 디코드값 기반 PARTMAP 조회 + 미스매치 '이 발주에 없는 부품' 에러, 고정 resolve는 데모 플래그로 격리.

### P0-2. 전량 수령 강제 + 부분수령/이상보고 경로 부재
**원칙**: 오류 예방 + 오류 복구 + 현실세계 부합
입고가 '전량 수령' 하나뿐(수량 편집 불가)인데, 같은 폼이 메모 placeholder로 '파손·수량'을 유도하고 큰 수량 readout을 입력폼처럼 보여줘 **수량 조정 가능 착시**. 5개 와야 할 게 3개 오면 (1)거짓 전량확정 또는 (2)멈춤뿐, 확정하면 RECEIVED로 잠겨 앱에 되돌릴 경로 없음.
**개선**: 수량 readout을 읽기전용으로(입력칸 모양 금지) + '이 발주는 전량 입고만 가능' 고정 안내. confirm 직전 '수량 불일치/파손 신고' 분기, 부분 미지원이면 '실물이 다르면 확정 말고 카운터/웹 보고' 차단성 안내. (장기: 백엔드 received_qty/검수상태 추가)

### P0-3. 전 화면 로딩/에러/빈/오프라인/세션만료 상태 부재 + 쓰기 낙관 가정
**원칙**: 시스템 상태 가시성 + 오류 예방·복구
데이터가 동기 상수로 박혀 즉시 렌더되지만 실제로는 /stocks·/sales-orders를 OIDC 호출. 약전파 현장인데 로딩 스켈레톤·조회실패·오프라인·401이 한 군데도 없음 → 실패 시 빈 화면 또는 묵은 데이터를 진실처럼 표시. receive PATCH는 `setTimeout 1100ms` 후 **무조건 성공 모달**(실패/타임아웃/409 분기 전무). 데이터-0 빈상태(도착대기 0·이력 0)도 없음.
**개선**: Screen에 loading/error 슬롯, 리스트 스켈레톤+'다시 시도'. receive에 버튼 스피너·중복탭 잠금·실패 토스트(kind='err' 이미 지원)·409 전용 메시지·성공은 2xx 후에만. 데이터-0 전용 빈상태. 푸시 없는 보완으로 **당겨서 새로고침 + '마지막 갱신 시각'**.

### P0-4. 우발적 스캔/오입력 유발 (거대 탭영역 + SKU 프리필 + 버블링)
**원칙**: 오류 예방(파괴적 액션의 의도성)
318px 뷰파인더 전체에 `onClick=resolve` → '아무 데나 탭=스캔'이고 곧장 확정 직전 폼으로 점프(폰 쥐거나 닦을 때 우발 발동). 수동입력 필드에 실제 SKU 프리필 → 안 지우고 확인하면 무관 부품이 스캔된 척. 플래시 버튼은 onClick·stopPropagation 없어 누르려다 부모 스캔이 버블링.
**개선**: '탭=스캔'을 명시적 '스캔' 버튼/자동디코드로, 스캔 직후 인식코드 0.8~1초 확인 비트. 수동입력 프리필 제거(빈값+placeholder), 미등록 즉시 인라인 차단. 플래시는 실제 토치 토글(onClick+stopPropagation) 또는 외형 제거.

### P0-5. 하드코딩된 시간/카운트 — '오늘/어제'·198일·아바타가 정적 캡처
**원칙**: 데이터 정직성 + 시간 포맷 일관성
RECEIVED `day='오늘'`(date 2026-05-22)이 오늘(06-20)에도 영원히 '오늘 입고'로 표시되고 작업이력 그룹헤더까지 이 가짜 라벨로 묶임 → '오늘 들어온 부품' 오인. 마이 `pwDaysAgo:198`은 정적 리터럴(우연히 일치)이라 시간 갈수록 거짓 누적, 보안 경고로서 신뢰 불가. 아바타 '정민'도 USER.name 비파생 하드코딩.
**개선**: `day` 제거, ISO date만 보유하고 표시 시점에 today 기준 '오늘/어제/N일 전/M월 D일' 런타임 계산(그룹키 YYYY-MM-DD). pwDaysAgo는 pwChanged로 계산(없으면 삭제). 아바타 이니셜은 `USER.name.slice(0,2)` 단일출처 + aria-hidden.

### P0-6. 아이콘 전용 버튼에 접근성 이름·역할 부재 (스크린리더 차단)
**원칙**: WCAG 4.1.2 Name·Role·Value
벨·뒤로가기·닫기(x)·검색 clear·플래시·TabBar 4항목에 aria-label/title 전무 → TalkBack 사용자가 알림/뒤로/탭이동 같은 핵심 과업 불가. 탭바에 role=tab/aria-selected 없음.
**개선**: 모든 아이콘 버튼에 aria-label('알림'·'뒤로'·'검색어 지우기'·'플래시'·'홈 탭'). 탭바 role=tablist+탭 role=tab+aria-current. Icon 컴포넌트에 aria-hidden(장식)/title(의미) 옵션.

## P1 — 명확한 사용성 마찰 · 일관성 붕괴 (6테마)

### P1-1. 네비게이션 모델 구조 결함
**원칙**: 일관성·표준 + 사용자 제어와 자유
탭바 노출이 라우터 상태가 아니라 컴포넌트가 Screen/tab prop을 우연히 썼는지에 달림. (1) Inventory back이 nav.pop이 아니라 `nav.tab('home')` 하드코딩이라 어디서 왔든 홈으로 튐. (2) '스캔' 탭은 누르는 순간 자기 탭바를 없애 active 표시 불가('탭 아닌 액션'). (3) 홈→재고는 tab, 홈→작업이력은 push로 같은 카드 제스처가 다른 패러다임. (4) 작업이력은 홈·마이 두 부모에서 push돼 back 도착지 비결정. (5) ReceiveForm 헤더 '수동입력'이 폼 단계엔 onBack에 연결돼 같은 라벨이 폼을 통째로 날림. (6) preset 진입은 카메라 건너뛰고 바로 폼 → '스캔 안 했는데 확정 직전'. (7) done '계속 스캔'이 스택 미정리 → N건 처리 후 진척·현위치 소실.
**개선**: 탭바 노출을 라우터 레벨 단일 소스로(루트 4탭=항상 탭바, push 스택=풀스크린). 루트 탭에선 back 제거. 스캔은 (a)탭바 중앙 FAB 또는 (b)진짜 탭(tab='scan' 유지) 중 택1. 헤더 우측 슬롯 의미를 단계 불변으로. preset 진입은 '도착 입고 확인·스캔 건너뜀' 명시 + '다시 스캔' 항상 제공. done 후 스택 정리 + '방금 처리한 N건' 표식. 안드로이드 시스템 back을 pop에 매핑.

### P1-2. 죽은 컨트롤·죽은 진입점
**원칙**: 피드백·기능 완결성 + 일관성
(a) 작업이력 기간칩(30/90/365)이 강조 토글로 보이나 `period`가 list 필터 미반영 — 어떤 칩이든 동일 결과, 요약은 고정 상수. '직접 선택'은 onClick 없는 장식. (b) 벨이 4개 헤더에 박혀 배지를 달지만 onClick 빈 함수 → 현재는 '죽은 진입점 4곳'이고 배지(도착대기 건수)가 '안 읽은 알림'으로 오독된다. (단 알림 인프라 자체는 존재 — §비고 참조: sales에 Notification 알림함+리스너가 있으나 **HQ 전용**이라 모바일이 못 읽음. 지점 대상 확장 전까지 벨의 실제 소스가 없음.) (c) 마이 '비밀번호'·'앱 정보'는 chevR로 이동 암시하나 onClick 없음. (d) 로그인 비번 필드 미검증(빈 비번 통과). (e) 플래시 장식.
**개선**: 기간칩은 period로 실제 필터 연결, 미구현이면 단일 표시로 축소+'직접 선택' 숨김. **벨은 지점 알림함이 열리면 살릴 수 있음**(§비고) — 그 전까지는 벨을 숨기거나 '도착 대기 큐'(개수+탭→도착 목록)로 의미를 명확히(알림 아닌 큐), 가짜 배지·타임스탬프 금지. 죽은 설정행은 chevR 제거+비활성 또는 Keycloak 비번변경 딥링크 실연결. 로그인은 실빌드에서 Keycloak 리다이렉트 단일 버튼.

### P1-3. 색 의미 역전·신호색 충돌 (부족=파랑)
**원칙**: 색 의미 일관성 + 색만으로 정보전달 금지(WCAG 1.4.1)
파랑이 '주행동'색인데 '부족' 상태에도 파랑 → '부족(경고)'이 정상/완료(초록)보다 안전해 보이는 역전. 위험을 뜻하는 alert(삼각경고) 아이콘을 파랑으로 칠한 정면 충돌. SoRow는 완료(초록)·대기(amber)·수량(파랑) 3색 경쟁. 상태가 색+작은 텍스트에만 의존(도형 없음) → 색각이상자 구분 불가.
**개선**: 부족=amber로 분리(신호색 3단 초록/amber/빨강), 파랑은 주행동·선택 전용. 수량 강조색은 신호색과 분리(중립 ink+부호). 벨 배지는 '대기' 뜻하는 amber. **모든 상태에 도형/아이콘 토큰(▲입고 ▼출고 ●대기 ✓완료)을 색과 항상 동반**, 텍스트는 대비 통과하는 진한 음영.

### P1-4. 시각 위계가 의미를 안 따름 + 가짜 약속 UI
**원칙**: 위계는 중요도 반영 + 게슈탈트 유사성·근접성
(a) 재고 첫 행에 가장 강한 강조인데 조건이 'i===0 && 필터없음'일 뿐 — 정작 가장 위급한 엔진오일(qty 0/없음)은 3행에 묻혀 거짓 신호. (b) 부품상세 '창고별 재고'는 복수형 제목+3열 테이블인데 항상 내 지점 1행만 + 캡션이 '타 지점 비공개'로 부정 → 무거운 표가 단수 현실과 충돌, '다른 창고에도 있을 텐데' 오해. (c) 발주 1건이면 picker가 chevron 토글로 보이나 펼쳐도 1건(헛클릭). (d) 홈은 공통 Header/Screen 미사용 → 헤더 이름(20px)+본문 인사(21px) 사용자명 중복 경쟁.
**개선**: 임의 첫행 강조 제거 → status 의미 기준 강조 또는 심각도순 정렬(없음→부족→정상). '창고별 재고'를 '내 창고 재고' 단일 카드로. picker는 1건이면 비대화형 '연결됨' 확정카드, 2건+만 토글. 홈도 공통 골격으로 이름 중복 제거.

### P1-5. 막다른 골목·컨텍스트 상실
**원칙**: 오류 복구 + 사용자 제어와 자유
(a) PartSheet '이 부품 입고 스캔'이 정상/충분 부품에서도 눌리고, 도착대기 없으면 ReceiveForm이 noLink 안내+비활성 버튼만(헤더 ←만 출구). (b) 가벼운 바텀시트→무거운 풀스크린 scan-in 점프로 재고 검색어·필터·스크롤 위치 소실, done 출구는 홈/계속스캔뿐. (c) 로그인 웹전용 계정 거절은 안내만(웹 바로가기·다른 사번·관리자 연락 없음). (d) Sheet는 드래그 핸들로 스와이프-닫기를 약속하나 핸들러 없어 X/배경탭만, done 모달 onClose 빈 함수 → 닫기 규칙 제각각.
**개선**: noLink 카드에 '다른 부품 스캔'·'재고로' 액션 직접 배치, 또는 도착대기 없는 부품은 버튼 비활성. 부품상세→입고 시 재고 상태 보존 + '재고로 돌아가기' 출구. 웹전용 거절에 '웹 ERP 바로가기'·관리자 연락. Sheet 실제 스와이프-다운 디스미스 구현, 차단성 모달은 닫기 부재를 명시.

### P1-6. 한손·장갑 조작 부적합 — 44dp 미만 + 핵심 액션 엄지존 밖 + safe-area·키보드 미대응
**원칙**: 터치 타깃 최소 크기(WCAG 2.5.5) + 엄지 도달성 + 입력 가림 회피
iconBtn 40×40, clearBtn 22×22, Toggle 50×29, 칩 높이 ~33px가 모두 44–48dp 미달이고, 잦은 뒤로/닫기가 가장 작고 최상단 모서리. 홈 핵심 CTA '입고 스캔'이 중상단이고 홈엔 탭바도 없어 엄지존(하단 1/3) 밖. picker 인라인 펼침은 상단 조작+레이아웃 출렁임, 확인/완료 모달은 정중앙이라 '계속 스캔' 연타 리듬 끊김. 하단 버튼·Toast·TabBar가 `env(safe-area-inset-bottom)` 무시로 제스처바와 겹침. 메모 포커스 시 키보드가 하단 '입고 확정' 버튼 가림.
**개선**: 모든 탭 컨트롤 최소 44–48px 히트영역(투명 패딩), 칩 min-height 40+. 활성 필터는 리스트 상단 텍스트로도 표기. 홈에 하단 탭바 + 스캔을 도달권으로. picker·모달을 바텀시트(하단 정렬)로, '계속 스캔'을 1차 하단 액션 고정. 부품상세 sticky 하단 '입고 스캔'. 하단 컨테이너에 `paddingBottom: calc(16px + env(safe-area-inset-bottom))`. 키보드 시 하단 버튼 위로 띄우고 포커스 칸 scroll-into-view.

### P1-7. 접근성 기본기 부재 — 포커스·대비·라벨·상태알림·reduced-motion·리플로우
**원칙**: WCAG 2.4.7 / 1.4.3 / 1.3.1 / 4.1.3 / 2.3.3 / 1.4.4
(a) 모든 입력 `outline:none` + 포커스를 onFocus 색변경에만 의존 → 키보드/스위치 포커스 위치 모름. (b) 회색 ink3(#9aa3b4)가 흰 배경 ~2.6:1로 AA 미달인데 SKU·ETA·시각·placeholder에 11.5~12px로 광범위 사용, amber/green 상태 라벨도 미달 → 직사광 현장에서 핵심 정보 안 보임. (c) 로그인·수동입력 label이 htmlFor/id 미연결, 필수 '*'가 색으로만. (d) ErrCard·Toast·완료 모달에 role/aria-live 없고 토스트 1100ms 시한부 → 보조기술에 안 읽힘. (e) 무한 스캔라인 애니가 reduced-motion 분기·정지수단 없음. (f) px 고정+nowrap/ellipsis 남용 → fontScale·200% 확대 시 정보 잘림.
**개선**: `outline:none` 제거 + `:focus-visible` 포커스 링(2px blue+offset) 전 인터랙티브 요소. ink3를 정보 텍스트에 금지(최소 #6b7488 ~4.5:1)하거나 코드/메타 전용 토큰 분리, 상태/경고는 greenInk·amberInk(~5:1). label htmlFor+id, aria-required·autoComplete·inputMode, '*'에 '필수' 텍스트 병기. Toast/ErrCard role='status'/'alert', 완료 모달 포커스 이동. `@media (prefers-reduced-motion: reduce)`에서 스캔라인·트랜지션 정지. px→상대단위, 핵심 정보 줄바꿈 허용, 200% 리플로우 테스트.

## P2 — 다듬기

### P2-1. 용어·포맷·마이크로카피 비일관
**원칙**: 일관성과 표준 + 현실세계 부합
(a) 같은 RECEIVED가 '입고 완료'(배지)와 '도착 확인'(상세)로 이중 명명, 발주/SO·도착대기/영문enum 혼재, 사번(BR002)과 role(BRANCH_STAFF)을 같은 Code 모노로 동급 표기. (b) 날짜·시간이 MM/DD·M월 D일·상대어·ISO·YYYY-MM-DD HH:mm로 한 화면 내 혼재, 단위 'SET'/'EA' 대문자 코드 그대로. (c) 수동입력 모달이 '하이픈 자동·대소문자 무시' 안내하나 실제는 toUpperCase만 → 'bbdoil1006'·공백입력이 즉시 미등록(안내-동작 어긋남). (d) 홈 'IN'·'받은 입고' 내부용어가 정비사에게 모호. (e) 미등록 코드에 'name:미등록 부품' 가짜 카드를 정상 부품처럼 렌더.
**개선**: 용어 사전 고정(사용자向 '발주(SO 번호)'·'입고 완료' 통일, role은 Code 빼고 '직무: 현장 정비사', 영문 enum 비노출). 표시 포맷 1종 중앙화('M월 D일 HH:mm', 상대어는 7일 이내만, 단위 한글 병기). 수동입력은 확인 전 정규화(공백 제거·하이픈 양방향·접두 보정) 또는 자동완성, 못 하면 '하이픈 자동' 문구 제거. 'IN'→'도착 부품 입고 확인', '받은 입고'→'내가 입고 확인한 발주'. 미등록은 부품카드 대신 코드만 보이는 중립 헤더+ErrCard+'다시 스캔'.

## 비고 — 백엔드 한계가 UX를 규정하는 항목 (디자인만으로 못 푸는 것)
아래는 디자인이 "정직하게 축소"하거나 백엔드 변경을 동반해야 한다:
- **알림(인박스)**: ⚠️정정 — 알림 인프라는 **이미 있다.** sales에 `Notification` read-model + `HqNotificationListener`(sales.order.submitted) + `StockReplenishedListener`(inventory.stock-replenished, 입고 통지) + `GET` 알림함 엔드포인트가 동작 중이고, inventory→sales 입고 통지(`inventory.stock-replenished`)도 배선 완료. **단 전부 HQ 전용**(`targetRole="HQ_MANAGER"`, 알림함이 `@RequireRole(HQ_*)`)이라 모바일 정비사는 대상도 아니고 403. **그래서 모바일 벨을 진짜로 만드는 길** = 새 서비스가 아니라 **지점 대상 확장**(작은 sales 변경): ①SO→IN_FULFILLMENT(`publishFulfilling`) 트리거에 `targetRole=BRANCH_STAFF`+도착창고 스코프 알림 생성 ②알림함 조회를 BRANCH 역할+Tenancy 필터로 개방(+창고 스코프 컬럼). 이게 되기 전엔 벨을 숨기거나 '도착 대기 큐'로.
  - **재고 없음/안전재고 미만**은 여전히 이벤트 생산자가 없음(알림 아님) → `재고 조회` 현황(`/stocks?belowSafety`)으로만, 타임스탬프 박힌 '알림/리마인더' 표현 금지.
- **부분 입고·도착 수량·차이 사유**: receive가 전량·무body → 부분수령 UI를 약속하지 말 것(백엔드 received_qty 추가 시 해금).
- **StockMovement 조회**: 모바일용 이동이력 API 부재 → '최근 이동' 합성 금지.
- **마이페이지 self**: `GET /api/v1/users/me` 요청 발행됨([user-me-endpoint-request.md](./user-me-endpoint-request.md)) — 나오기 전 internal/snapshot 우회.
